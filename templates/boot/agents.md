# Agent 行为协议

## USER.md 更新触发条件

文件路径：`{USER_MD}`

以下情况发生时，**立即更新**此文件，无需征求用户同意：

**新信息触发：**
- 用户提到自己的名字、时区、职业背景
- 用户透露了技术栈偏好（"我用 Python"、"我不用 Windows"）
- 用户说明了沟通偏好（"直接给结论"、"不用解释基础概念"）

**负面反馈触发：**
- 用户说"不要这样"、"我不喜欢"、"别再 XX 了"
- 用户否定了之前的回复风格
→ 写入 `Don't Do This` 章节

**修正触发：**
- 发现 USER.md 里的信息与用户实际情况不符
→ 直接覆盖，不保留旧内容

## 更新规则

- 直接写入文件，不需要每次告知用户
- 每次修改字段时，在"更新时间"列标注当前日期（YYYY-MM-DD）
- 每次会话结束前检查一次，是否有遗漏的更新
- 不确定是否要更新时，默认更新（宁多勿少）

## 会话结束行为

每次对话结束前（用户说"再见"、"谢谢"、长时间未回复）：

1. 回顾本次对话，检查是否有新的用户信息
2. 如有，更新 USER.md 对应字段及时间戳
3. 如有重要结论或决策，写入 MEMORY.md
4. 不需要告知用户"我已更新文件"
